# VimQuest — Vim 학습 RPG 게임 기획서

> 버전 0.1 (초안) · 작성일 2026-06-26
> 한 줄 요약: **"젤다처럼 맵을 탐험하며, 키를 모으고, Vim 명령어로 퍼즐을 풀어 자연스럽게 Vim을 체득하는 무료·한국어 지원 브라우저 게임"**

---

## 1. 비전 & 문제 정의

### 1.1 우리가 푸는 문제
- Vim 입문 자료는 대부분 **건조하고 어렵다.** 명령어를 한꺼번에 외우게 강요해서 **흥미가 먼저 떨어진다.**
- 가장 재밌는 기존 게임(VIM Adventures)은 **유료($35/6개월)**, **한국어 미지원**, **한글 IME 상태에서 작동 불가**.
- 무료 대안들은 미완성이거나(VimGore), 단순 드릴이거나(VimGenius), 속도 훈련 전용(Vim Racer)이라 **"입문자가 재밌게 처음 배우는" 자리가 비어 있다.**

### 1.2 우리의 해법
플레이어를 가르치려 하지 않는다. **놀게 한다.**
- Vim의 핵심 통찰("명령어 = 동사 + 이동/대상")을 **게임 메커니즘 자체로 체득**시킨다.
- 명령어를 한꺼번에 주지 않고, **레벨마다 1~3개씩** 해금하며 점진적으로 쌓는다.
- **무료 + 한국어 우선 + 한글 IME 자동 처리**로 기존 게임의 빈자리를 정확히 노린다.

### 1.3 핵심 차별점 (경쟁 비교)
| | VIM Adventures | Vim Racer | **VimQuest (우리)** |
|---|---|---|---|
| 형태 | 젤다식 RPG | 속도 챌린지 | **젤다식 RPG** |
| 가격 | 유료 $35 | 무료 | **무료** |
| 한국어 | ❌ | ❌ | **✅ 우선 지원** |
| 한글 IME | 작동 불가 | - | **자동 감지·안내** |
| 입문자 친화 | 보통 | 낮음(중급용) | **매우 높음** |
| 점진적 해금 | ✅ | ❌ | **✅** |

## 2. 타겟 사용자

- **1차:** Vim을 배우고 싶지만 기존 자료가 어려워 포기했던 한국어권 개발자 / 학생
- **2차:** VS Code·터미널은 쓰지만 Vim 모션은 모르는 주니어 개발자
- **사용 맥락:** 짧은 세션(5~15분)으로 출퇴근·휴식 시간에 한 레벨씩. "공부"가 아니라 "게임"으로 인식되게.

**비목표(Non-goal):** Vim 고수를 위한 고급 매크로/플러그인 마스터리. (그건 VimGolf의 영역. 우리는 **입문~중급 진입**에 집중.)

## 3. 핵심 게임 컨셉

### 3.1 한 문장 컨셉
> 당신은 텍스트 세계 **Textland**에 떨어진 **커서(@)**. 잃어버린 키보드 키들을 되찾고 막힌 문을 열며 세계를 탐험한다. 각 키는 새로운 Vim 능력이 된다.

### 3.2 핵심 루프 (Core Loop)
```
맵 진입 → 새 명령어 1~2개 안내(짧게) → 그 명령어로만 풀리는 퍼즐/이동
   → 키 아이템 획득(능력 해금) → 문 열림 → 다음 구역 → 보스 챌린지(타임어택)
```
- **배움 → 즉시 사용 → 보상 → 반복.** 명령어를 배운 직후 그것 없이는 못 지나가는 구간이 나와서 **반드시 손으로 쓰게** 된다. (PacVim·VIM Adventures의 검증된 패턴)

### 3.3 핵심 메커니즘: "커서가 곧 캐릭터"
- 플레이어는 텍스트 그리드 위의 **커서**. 실제 Vim처럼 **모드(Normal/Insert)** 가 있다.
- 이동·편집이 전부 **실제 Vim 키**와 1:1 대응. 게임을 잘하게 될수록 = Vim을 잘하게 됨.

## 4. Vim 명령어 → 게임 요소 매핑

게임의 모든 장애물은 "특정 명령어를 써야만" 풀리도록 설계한다.

| Vim 개념 | 게임 표현 | 학습 효과 |
|---|---|---|
| `h j k l` | 4방향 이동(상하좌우 발판) | 기본 이동 반사 |
| `w b e` | 단어 단위 점프 — 멀리 떨어진 발판은 한 칸 이동으론 못 감 | 단어 모션 |
| `0 $ ^` | 줄의 끝/처음에 있는 보물/문 | 줄 이동 |
| `gg G` | 위층/아래층 워프 포털 | 파일 이동 |
| `f{char} t{char}` | "이 글자까지 순간이동" 레이저 발판 | 검색 점프 |
| `x` | 길을 막은 장애물 글자 1개 제거 | 삭제 |
| `dw d$ dd` | 막힌 단어/줄 통째로 제거 (동사+모션 조합!) | **문법 체득** |
| `i a o` | Insert 모드로 빈칸에 글자(열쇠말) 채워 문 열기 | 모드 전환·삽입 |
| `ciw ci(` | 괄호 안 함정 내용물 교체 | 텍스트 객체 |
| `.` (반복) | 같은 함정 여러 개를 한 번 배우고 연타 | 반복의 힘 |
| `u Ctrl-r` | 되감기 룬 (실수 복구) | 안전망 |
| `/{pattern} n` | 숨겨진 보물 위치 검색 | 검색 |

> **핵심 설계 원칙:** 단순 키 입력이 아니라 **`동사 + 이동` 조합(`dw`, `ci(`)** 을 풀어야 통과하는 퍼즐을 중반부에 배치해서, 플레이어가 "Vim은 언어다"를 스스로 깨닫게 한다.

## 5. 월드 & 레벨 구조

### 5.1 월드 구성 (전체 로드맵 기준)
점진적 학습 4단계(생존→편안함→빨라짐→초능력)를 4개 월드로 매핑:

| 월드 | 테마 | 핵심 명령어 | 목표 |
|---|---|---|---|
| **W1. 이동의 숲** | 기초 이동 | `hjkl`, `w b e`, `0 $` | "방향키 없이 다닐 수 있다" |
| **W2. 점프 협곡** | 빠른 이동 | `gg G`, `f t`, `/ n` | "목표로 바로 점프한다" |
| **W3. 편집의 던전** | 삭제·삽입 | `x`, `dw dd`, `i a o`, `.` | "동사+모션 문법을 쓴다" |
| **W4. 텍스트 객체 신전** | 고급 편집 | `ciw ci(`, `yy p`, visual | "구조를 편집한다" |

- 각 월드 = 5~8개 레벨. 각 레벨 = 5~15분.
- 월드 끝마다 **보스전 = 타임어택 챌린지**(그 월드 명령어를 제한시간 안에 종합 사용). 별 1~3개 평가.

### 5.2 레벨 1 예시 (튜토리얼) — "첫 발걸음"
```
┌─ Textland: 이동의 숲 1-1 ──────────┐
│                                    │
│   @ . . . # . . . . [열쇠] . . $   │
│                ▲                   │
│           바위(x로 제거)            │
│                                    │
│  목표: 'w'키를 주워 멀리 점프 해금   │
│  지금: hjkl로 이동해보세요          │
└────────────────────────────────────┘
```
- 입력: `l`을 눌러 오른쪽 이동 → 바위(`#`) 만남 → 힌트 "`x`로 제거" → 통과 → `[열쇠]` 도달 시 `w` 능력 해금 → `$`(줄 끝)의 문으로.

## 6. 화면 / UI 구성

```
┌──────────────────────────────────────────────┐
│  ❤❤❤   World 1-3   ⏱ 04:12   ⭐⭐☆         │ ← 상단 HUD
├──────────────────────────────────────────────┤
│                                              │
│              [ 게임 타일맵 영역 ]              │ ← 메인 캔버스
│                                              │
├──────────────────────────────────────────────┤
│ 모드: NORMAL │ 마지막 입력: dw │ 힌트: [?]    │ ← 하단 상태바(실제 Vim 흉내)
└──────────────────────────────────────────────┘
   [해금된 키] h j k l w b  ·  cheat sheet 토글: Ctrl+/
```
- **모드 표시줄**을 실제 Vim처럼 항상 노출 → Normal/Insert 감각을 몸에 익힘.
- **입력 에코**: 방금 누른 키 시퀀스를 화면에 표시(`dw`처럼) → 자기가 뭘 쳤는지 인지.
- **치트시트 오버레이**(Ctrl+/): 막히면 켜고, 익으면 끄는 식으로 의존도 감소.

## 7. MVP 범위 (첫 목표 = Vertical Slice)

"끝까지 한 줄로 동작하는 가장 작은 게임"을 먼저 만든다.

### ✅ MVP에 포함
- [ ] 타일맵 렌더링 (격자 + 커서 + 벽 + 문 + 키 아이템)
- [ ] **W1 이동의 숲 3개 레벨** (`hjkl`, `w b e`, `x`, 문/열쇠)
- [ ] 모드 표시줄 + 입력 에코 + 체력/실패·재시작
- [ ] 키 획득 → 능력 해금 → 다음 레벨 진행
- [ ] 한국어 UI + **한글 IME 감지 → "영문 입력으로 전환하세요" 안내**
- [ ] 치트시트 오버레이(Ctrl+/)
- [ ] 레벨 데이터 = 외부 파일(JSON/텍스트맵)로 분리 → 레벨 추가가 코딩 없이 가능

### ❌ MVP 제외 (이후 단계)
- 계정/로그인, 리더보드, 진도 클라우드 저장 (→ localStorage로 임시)
- 사운드·애니메이션 폴리시, 스토리 NPC 대사
- W2~W4, 보스 타임어택, 매크로 레벨
- 모바일 터치 대응

### MVP 완료 기준 (Definition of Done)
> 처음 보는 사람이 설명 없이 접속해서, **방향키를 한 번도 안 쓰고** hjkl·w·x만으로 3개 레벨을 클리어할 수 있다. 한글 키보드 상태로 들어와도 막히지 않는다.

## 8. 기술 아키텍처 (검토안)

플랫폼은 **웹 브라우저** 확정. 구현 방식 2안 — **MVP는 A안 권장.**

### A안. 풀 Go (Ebitengine → WebAssembly) ⭐ MVP 권장
```
Go + Ebitengine ──build──▶ game.wasm ──▶ <canvas>
```
- **장점:** 게임 로직(키 시퀀스 파싱·충돌·상태머신)이 Go라 깔끔·테스트 용이. 같은 코드로 **데스크톱(.app/.exe) 빌드 덤**. 타일맵/입력 기본 지원.
- **단점:** WASM 번들 2~10MB(첫 로딩 느림), 한글은 **Noto Sans KR TTF 임베드**(`text/v2`) 필요, HUD/버튼도 캔버스에 직접 그림.
- **적합:** Go를 선호하고, 로직 견고함 + 멀티플랫폼을 원할 때.

### B안. 하이브리드 (JS 프론트 + Go 백엔드)
```
프론트: Canvas/JS(작고 빠름) ── API ──▶ Go 백엔드(검증·리더보드·진도)
```
- **장점:** 웹 로딩 가볍고 DOM UI 자유로움. Go의 강점은 서버에서 활용.
- **단점:** 코드베이스 2개, MVP엔 과함. (리더보드·계정 붙일 때 가치)

### 권장 경로
> **MVP는 A안(풀 Go/Ebiten)** 으로 빠르게 동작시키고, **계정·랭킹이 필요해지는 시점에 B안의 Go 백엔드를 추가**하는 단계적 진화.

### 레벨 데이터 포맷(예시 — 코딩 없이 레벨 추가)
```
# level_1-1.txt
@.....#.....K....$
# 범례: @=시작 .=바닥 #=바위(x) K=열쇠('w'해금) $=출구
meta: { unlock: "w", hint: "hjkl로 이동, #는 x로 제거" }
```

## 9. 한글 / 접근성 (필수 차별점)

- **한글 IME 감지:** 입력 이벤트가 조합문자(`isComposing`)거나 예상 키와 다르면 → 화면에 "🇰🇷→🇺🇸 영문 입력으로 바꿔주세요" 토스트. (기존 게임들의 최대 불만 해소)
- **모든 UI·힌트·튜토리얼 한국어.** 영어 원문 명령어는 그대로(어차피 키는 영문).
- **색맹 고려 타일 팔레트**, 키보드 전용 조작(마우스 불필요 — Vim 철학과 일치).

## 11. 성공 지표 (KPI)

- **활성화:** 접속자 중 레벨 1-1 클리어 비율 (목표 ≥ 70%)
